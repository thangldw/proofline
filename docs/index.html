Why and how

Fluent answers can hide missing evidence. Trace every claim.

Proofline preserves provenance across ingestion, retrieval, learning artifacts, and human-reviewed AI proposals. The deterministic offline core still works when no model is configured.

  1. 01 · CAPTUREDocuments, notes, and local GitImport approved sources with stable identities instead of copying context into another silo.
  2. 02 · VERSIONImmutable source revisionsEdits create new versions. Historical evidence remains resolvable and inspectable.
  3. 03 · RECOVERDecisions and grounded answersSearch rationale, timelines, contradictions, memories, and learning artifacts.
  4. 04 · VERIFYExact lines and quote hashesOpen the source identity, revision, offsets, and exact span behind each derived claim.

Offline core: ingestion, versioning, lexical retrieval, memory review, study, Studio, backup, and integrity checks. Optional providers: Qwen, DeepSeek, Ollama, vLLM, and OpenAI-compatible endpoints with explicit egress.

Known limits

Evidence stays honest about what it proves.

Proofline is a public experimental pre-alpha. It narrows uncertainty, but it does not turn mock fixtures or one platform receipt into production truth.

macOS is not Windows

The installed receipt covers Darwin ARM64 only; a real Windows lifecycle run remains open.

Wheel is not native packaging

The signed native installers (.app, .dmg, or .exe), updater, and application rollback channel are not yet qualified.

Mock is not model quality

Scripted provider fixtures verify wiring, not extraction or grounded-answer quality.

Tests are not production support

External pilot, security qualification, ownership, and operational commitments remain open.
