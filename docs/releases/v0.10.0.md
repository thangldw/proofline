# Proofline v0.10.0

Proofline v0.10.0 adds the local runtime lifecycle needed by a future macOS or Windows wrapper. It
does not claim that a signed desktop app or production distribution is complete.

## Added

- `proofline serve --port 0` binds an OS-selected loopback port and emits structured readiness only
  after application startup succeeds.
- `--ready-file` publishes the same metadata atomically with owner-only permissions and removes it
  after graceful shutdown.
- `--data-dir` owns the default SQLite database and provider configuration through
  `PROOFLINE_HOME`, while the existing explicit database/config environment variables still win.
- `--web-dir` serves a built Proofline web archive from the API process, giving native wrappers one
  same-origin process to supervise.
- `/health` now identifies the running Proofline version.

## Quality

- A subprocess integration test proves dynamic-port startup, database creation, same-origin web and
  API health, `SIGTERM` shutdown, zero exit, and ready-file cleanup.
- The current Search → Sources → Settings flow was captured and audited at 1280 × 720 with no
  browser warnings or errors. The Sources screen retains an accessibility follow-up for semantic
  row/column relationships.
- A source-backed quality report confirms that all six checked-in evaluation receipts qualify their
  claims, while identifying incomplete timestamp and tested-revision metadata.

## Limitations

- Windows verification, Docker qualification, signed installers, auto-update rollback, native
  launch integration, and production support remain open roadmap gates.
- The web archive remains a separate release artifact; v0.10.0 defines how a wrapper serves it but
  does not build a native wrapper.
