# Proofline v0.14.11

Proofline v0.14.11 completes the documentation needed to present the repository honestly as a
public experimental pre-alpha project without implying production support.

## Support policy

The root `SUPPORT.md` defines the supported single-user macOS experiment, latest-pre-release-only
policy, safe content-free issue reports, data-loss escalation, operator backup responsibility,
upgrade/database rollback and the absence of an SLA or fixed release schedule.

`MAINTAINERS.md` names the current repository maintainer and explicitly leaves production signing,
desktop updater, incident response, encrypted-retention operations, real-model dataset stewardship
and pilot support unassigned. Those gaps remain blockers rather than implicit commitments.

## Qualification boundary

The GitHub repository remains private until the owner explicitly changes visibility. This release
does not qualify Windows, signed native installers, application update rollback, encrypted backup
operations, real-model quality, external-pilot utility, security assessment or production support.

## Validation

- Public policy and roadmap consistency review.
- Full backend/frontend, lint, formatting, web build, egress and deterministic evaluation gates.
- Installed-wheel lifecycle, portability, backup restore/rollback, integrity and macOS Keychain
  receipt.
