# Proofline v0.14.7

Proofline v0.14.7 prepares the repository for an accurate public experimental pre-alpha launch and
introduces a durable gate matrix for the path to production.

## Public hygiene

- Remove contributor-machine absolute paths from Studio design QA.
- Update the current supported pre-alpha version in the security and support boundary documents.
- Keep the production exclusions explicit instead of presenting planned behavior as implemented.
- Ensure local release publication marks every `v0.x` build as a GitHub pre-release.

## Portability documentation

The README and recovery guide now document both supported schema-v1 import paths:

- exact-ID restore into an empty initialized database;
- explicit content-free preview followed by deterministic all-ID remapping into a non-empty target.

Existing target rows are never overwritten or deleted, and a stale preview or failed provenance
check rolls back the complete merge.

## Production boundary

The new production-readiness matrix defines the first production target as a local single-user
desktop distribution. Windows verification, signed native installers, secure OS credential
storage, real-model evaluation, external pilot evidence, operational ownership and security
qualification remain open gates.

## Validation

- Release metadata consistency checks.
- Backend and frontend suites.
- Python lint/format, TypeScript build, egress check and bundled-web parity.
- Deterministic extraction, retrieval and grounded-QA evaluation gates.
