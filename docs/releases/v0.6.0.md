# Proofline v0.6.0

`v0.6.0` improves retrieval quality and makes its performance boundaries inspectable.

## Reranking

- A provider-neutral interface supports deterministic local reranking and HTTP cross-encoders.
- Reranking runs only when explicitly requested and is applied after reciprocal-rank fusion.
- Search diagnostics expose rerank rank, score, and channel; the default remains unchanged.
- The synthetic token-overlap receipt reaches MRR 1.0 on three regression queries. This is not
  real-model or pilot evidence and is not sufficient to choose a production default.

## Grounding assessment

Answer statements expose deterministic `supported` or `uncertain` assessment. Direct statements
whose negation conflicts with evidence fail closed and use the bounded repair path. Synthesis and
inference polarity ambiguity is labeled uncertain instead of being overclaimed as contradiction.

## Vector candidate index

Semantic retrieval now queries locality-sensitive sign-vector bands before exact cosine scoring;
it no longer loads every stored JSON vector. Migration 16 backfills existing embeddings. Integrity
verification and source deletion include index rows.

The checked-in 1,000-source synthetic receipt records index time, search latency, peak Python
memory, database size, and no-op update cost. It explicitly does not qualify 10,000 files or 1 GB.
