# Proofline v0.5.0

`v0.5.0` adds local provider configuration and explicit model reliability workflows.

## Provider settings

- Configure Qwen, DeepSeek, Ollama, vLLM, or a generic OpenAI-compatible generation endpoint.
- Configure Ollama, vLLM, or a generic OpenAI-compatible embedding endpoint independently.
- API keys are stored only in the local owner-readable provider file and are never returned by
  the configuration or diagnostics APIs.
- Remote model egress remains disabled until explicitly enabled. Proofline never switches from a
  local provider to a remote provider automatically.

## Reliability

- Generation, embedding, and reranking expose separate capability states.
- The web console shows degraded mode while deterministic ingestion and lexical search remain usable.
- Transient transport failures receive at most three attempts. Exhaustion creates an inspectable
  `dead_letter` model run.
- A failed/dead-letter extraction run may be retried only against the same immutable source hash,
  provider ID, and model ID; the new run records manual-retry lineage.

## Boundaries

Reranking remains disabled until a benchmarked implementation is selected. The health endpoint
reports that capability honestly rather than implying it is available.
