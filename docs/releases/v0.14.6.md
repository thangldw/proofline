# Proofline v0.14.6

Proofline v0.14.6 makes the existing mock-qualified model-comparison integration runnable from
the CLI without QwenCloud, Ollama or another model service.

## Added

- Versioned scripted extraction and grounded-QA fixtures.
- A frozen local/remote mock comparison plan.
- Explicit `--allow-mock` support for both `eval-real-model-preflight` and `eval-real-model`.

## Evidence boundary

The scripted provider runs in-process and does not call either configured endpoint. Mock
preflight proves fixture identity and comparison wiring, not endpoint readiness. Comparison
receipts are always labeled `mock_integration`; perfect fixture scores are not real-model quality,
cost, latency or pilot evidence.

## Run locally

```bash
export PROOFLINE_MOCK_QWEN_API_KEY=mock-api-key
proofline eval-real-model-preflight \
  --plan evals/real-model/comparison-mock-v1.json \
  --output /tmp/proofline-mock-preflight.json \
  --allow-mock
proofline eval-real-model \
  --plan evals/real-model/comparison-mock-v1.json \
  --output /tmp/proofline-mock-comparison.json \
  --allow-mock
```

The fake credential exists only to exercise the credential-presence boundary and is never written
to either receipt.
