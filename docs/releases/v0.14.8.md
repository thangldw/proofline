# Proofline v0.14.8

Proofline v0.14.8 turns the local installed-package release gate into a durable, checksummed
platform qualification receipt.

## Receipt contents

The receipt binds the exact wheel name, size and SHA-256 to:

- Proofline version and Git revision;
- operating system, architecture and Python version;
- installed server start, readiness, bundled same-origin UI and graceful shutdown;
- immutable source revision and exact-evidence behavior;
- portable export, verification and empty-database import;
- SQLite backup verification and semantic integrity checks.

The receipt is generated from the clean release commit and uploaded beside the wheel, source
archive, web archive and `SHA256SUMS`.

## Qualification boundary

A receipt qualifies only the OS and architecture on which it ran. The macOS receipt does not
establish Windows behavior, native application signing, model quality, external pilot outcomes or
overall production support.

## Validation

- Backend and frontend suites.
- Python lint/format, TypeScript build, egress check and bundled-web parity.
- Deterministic extraction, retrieval and grounded-QA evaluations.
- End-to-end receipt generation through an installed Proofline executable.
