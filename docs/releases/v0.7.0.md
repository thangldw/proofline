# Proofline v0.7.0

`v0.7.0` introduces local workspace isolation and multi-worker-safe folder scan coordination.

Every workspace-scoped request uses `X-Proofline-Workspace-ID`. Omitting the header selects the
backward-compatible local workspace. Create and discover workspaces with `POST /api/v1/workspaces`
and `GET /api/v1/workspaces`.

Workspace identity now travels through ingestion, source inventory, search, grounded answers,
decision memory, model runs, audit events, Git ingestion, embedding indexing, and deletion. Reads
and mutations return `404` when an object belongs to another workspace.

Folder scans acquire an expiring SQLite lease before reading files. A second API worker receives
the explicit `scan_in_progress` failure instead of racing the active scan. This remains local
coordination; shared/team authorization and hosted synchronization are not implemented.
