# Proofline v0.14.17

Proofline v0.14.17 adds honest, target-specific qualification tooling for the two external gates
that can be prepared without fabricating their results. It remains an experimental pre-alpha.

## Real-Windows workflow

- `scripts/release_windows.ps1` runs the credential-free regression/evaluation suite on real
  Windows, builds wheel + PyInstaller sidecar + MSI + NSIS, qualifies the installed wheel and
  Credential Locker, writes checksummed receipts and publishes directly through GitHub CLI.
- The workflow uses a `[skip ci]` commit and never invokes GitHub Actions.
- `windows_desktop_receipt.py` refuses non-Windows hosts. A macOS cross-build cannot close the gate.

## Private pilot analyzer

- `scripts/analyze_pilot.py` requires a frozen private manifest with exact SHA-256 values.
- It rejects synthetic rows, version mismatches, duplicate IDs, missing foreign keys and tampered
  artifacts before calculating aggregate product/adoption/commercial metrics.
- Output contains only aggregates. Raw questions, identities, source content, answers and citation
  payloads stay in the access-controlled pilot workspace.
- Security qualification is explicitly recorded as `not_run_by_request` and therefore cannot
  support a production-ready claim.

## Boundaries

This release does not contain a Windows receipt or external-pilot outcome. Those require a real
Windows machine and consented participants. Real-model comparison remains excluded by the current
request; connector expansion and Team Brain remain gated by pilot evidence and authentication/RBAC.
