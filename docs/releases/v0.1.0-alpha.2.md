# Proofline v0.1.0-alpha.2

This pre-alpha release hardens Proofline's release and artifact-provenance path. Application
capabilities are otherwise the same evidence-first local vertical slice introduced in
`v0.1.0-alpha.1`; this is not a stable or production-supported release.

## Release hardening

- Tagged builds run all repository quality commands from an isolated project virtual environment.
- The packaged wheel is installed and seeded again in a separate clean environment before any
  artifact is published.
- Secret scanning is pinned to the resolved tagged commit, including manual immutable-tag rebuilds.
- A tag must resolve to a commit contained in `main`; recovery never moves or rewrites an existing
  tag.
- Read-only build/test permissions are separated from the narrowly scoped GitHub release publish
  permission.
- Version metadata, dated changelog entry, release notes, and lockfile version must agree with the
  tag before the build can continue.

## Assets

- Python wheel and source distribution for the local API/CLI.
- Optimized, unhosted web application archive for integrators. It requires a same-origin static
  host/reverse proxy that forwards `/api` and `/health` to the Proofline API.
- `SHA256SUMS` covering every uploaded build artifact.

Verify downloaded files before use:

```bash
sha256sum --check SHA256SUMS
```

Run the API/CLI wheel in an isolated environment:

```bash
python3 -m venv .venv-proofline
.venv-proofline/bin/pip install ./proofline-0.1.0a2-py3-none-any.whl
.venv-proofline/bin/proofline --version
.venv-proofline/bin/proofline seed
.venv-proofline/bin/proofline serve
```

The API binds to `127.0.0.1:8000` by default. Do not expose it to an untrusted network.

## Important limitations

- The API has no authentication or multi-user authorization.
- Repository Security plugin review, real-model comparison, external pilot metrics, Windows
  verification, reranking, scalable vector indexing, and production qualification remain open.
- Portable JSON import, desktop packaging, cloud sync, and production support are not included.
- The web archive is an integrator artifact, not a standalone server.

See the root README, security policy, changelog, and `v0.1.0-alpha.1` notes for the complete product
scope and data-handling limitations.
