# Proofline v0.1.0-alpha.5

This experimental pre-alpha release adds a deterministic integrity gate for live SQLite provenance
and closes an evidence gap for generated inference statements. It is not stable or
production-supported.

## Highlights

- `proofline verify-integrity` checks the configured live SQLite database without repairing or
  intentionally changing it.
- The verifier checks SQLite structure, foreign keys, complete migration history, stable source
  identity, latest immutable versions, content hashes and lengths, exact chunk spans and ordinals,
  governed-memory ownership, model-run lineage, exact evidence spans/quotes/hashes, embedding
  ownership/vector validity, and one-to-one FTS rows.
- Successful output contains only aggregate counts. Failures exit nonzero with a stable code and do
  not include source text, titles, paths, prompts, credentials, or corrupted values.
- Every generated statement—including an `inference`—must cite one or more server-issued evidence
  IDs. A missing citation receives one bounded repair attempt and then fails grounding validation.
- Release wheel smoke tests now run integrity verification both before portable export and after
  transactional import.

## Verify a database

```bash
PROOFLINE_DATABASE_URL=sqlite:////absolute/path/proofline.db \
  proofline verify-integrity
```

Use `verify-backup` first for a backup artifact, restore it according to the recovery guide, and
then use `verify-integrity` for Proofline's semantic provenance invariants.

## Assets

- Python wheel and source distribution for the local API/CLI.
- Optimized, unhosted web archive requiring a same-origin reverse proxy for `/api` and `/health`.
- `SHA256SUMS` covering every uploaded build artifact.

## Important limitations

- Verification detects known invariants but is not automatic repair, a security audit, or proof
  that source statements are factually true.
- Run verification while application writers are stopped when using it as an operational restore
  gate; it does not coordinate with another API process.
- The API has no authentication or multi-user authorization and binds to loopback by default.
- Repository Security plugin review, real-model comparison, external human pilot metrics, Windows
  verification, reranking, scalable vector indexing, and production qualification remain open.
- Desktop packaging, cloud sync, and production support are not included.

See the root README and backup/recovery guide before using approved data.
