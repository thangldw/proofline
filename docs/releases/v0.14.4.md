# Proofline v0.14.4

Proofline v0.14.4 completes the executable comparison runner while preserving a hard boundary
between mock integration results and real-model evidence.

## Added

- Run each preflighted provider through production memory extraction and grounded QA.
- Measure overall and per-kind extraction precision/recall/F1, exact citation resolution,
  insufficient-evidence abstention, latency, tokens and estimated cost.
- Continue evaluating other providers when one provider fails, while recording a sanitized failure
  code instead of prompt or source content.
- Write atomic comparison receipts classified as either `real_model` or `mock_integration`.

## Mock validation

The integration corpus uses a fake API key and injected provider transport. It covers one expected
memory, one negative source, one grounded answer and one required abstention. Both mock provider
profiles complete the pipeline, but their 100% fixture scores are wiring/calculation checks only.

## Run with real providers

```bash
proofline eval-real-model \
  --plan evals/real-model/comparison-v1.json \
  --output evals/real-model/receipts/comparison.json
```

The plan must pass preflight first. Mock manifests are rejected by normal real-provider transport.

## Qualification boundary

No real Ollama/Qwen/DeepSeek request was available for this release. Mock metrics must not be used
to select a production model or satisfy the roadmap's real-model and external-pilot gates.
