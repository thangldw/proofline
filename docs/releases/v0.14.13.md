# Proofline v0.14.13

Proofline v0.14.13 adds a platform-aware launcher to the installed Python wheel. It provides a
faster local app experiment without presenting the wheel as a native desktop application.

## Launch

After installing the wheel, run:

```bash
proofline launch
```

The command binds to `127.0.0.1` on a dynamic port, serves the bundled UI and opens the URL in the
default browser. It stores the SQLite database, provider configuration and temporary readiness file
under the operating system's application-data directory. `--data-dir` explicitly overrides that
location, and `--no-browser` leaves URL opening to the caller.

On macOS and Windows the launcher selects OS-backed provider-secret storage unless the operator has
already chosen another `PROOFLINE_SECRET_STORE` mode. Browser-open failure is visible but does not
stop the local server; the printed URL remains available.

## Qualification boundary

This is an installed-wheel launcher, not a signed `.app`, `.dmg`, `.exe` or `.msi`. It does not
close Windows lifecycle, native installer, signing, updater rollback, real-model, external pilot,
security or production-support gates.

## Validation

- Deterministic macOS, Windows and XDG application-data path tests.
- Subprocess launch, dynamic loopback readiness, bundled health, owned SQLite state and graceful
  ready-file cleanup.
- Browser success/failure behavior without making browser availability a server dependency.
- Full backend/frontend, lint, formatting, web build, egress and deterministic evaluation gates.
- Installed-wheel lifecycle, portability, backup restore/rollback, integrity and macOS Keychain
  release receipt.
