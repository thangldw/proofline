# Proofline v0.14.14

Proofline v0.14.14 extends the inspectable portable JSON archive to the evidence-first learning and
Studio surfaces introduced after schema v1. It remains an experimental pre-alpha release.

## Added

- Portable schema v2 records workspace scope and exact chunk identities.
- Study cards preserve immutable answer spans, scheduling state and append-only review history.
- Grounded action proposals preserve their model run, review status and exact chunk citations.
- Studio artifacts preserve deterministic content and ordered exact source citations.
- Empty-database restore and explicit non-empty merge cover every new collection atomically.

## Compatibility and integrity

The importer upgrades a verified schema-v1 core archive in memory before applying the schema-v2
contract. Schema-v2 verification rejects invalid workspace ownership, broken model/chunk/source
references, altered exact spans, quote-hash mismatches, duplicate citation ordinals and inconsistent
study review history. Merge continues to require a content-free preview digest and never overwrites
existing rows.

Embeddings, provider credentials, prompts and private staged retry inputs remain excluded. FTS rows
are rebuilt from the exported exact chunks; embeddings require explicit regeneration.

## Qualification boundary

This release does not add a real-model quality receipt, a Windows lifecycle receipt, a signed native
installer, external-pilot evidence, encrypted retention qualification or production support. The
existing macOS installed-wheel lifecycle remains the only qualified platform profile.
