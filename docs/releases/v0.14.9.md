# Proofline v0.14.9

Proofline v0.14.9 adds an operating-system-backed home for model provider API keys while preserving
the credential-free local development path.

## Provider secret storage

- Set `PROOFLINE_SECRET_STORE=os_keyring` before starting a desktop installation to use macOS
  Keychain or Windows Credential Locker.
- Generation and embedding keys can be replaced or explicitly removed from Settings and are never
  returned by the configuration API.
- Saving in keyring mode migrates legacy keys out of `providers.json`. Provider validation failure
  rolls back both non-secret configuration and keyring changes.
- The default `file` mode remains available for local development and stores the configuration with
  owner-only permissions.

## Platform receipt

The local release receipt now exercises set, read and delete against the installed OS-keyring
backend without recording the temporary value. The v0.14.9 macOS receipt qualifies only that
artifact and machine family; it does not establish Windows behavior.

## Qualification boundary

This remains an experimental pre-alpha release. It is not a signed native application and does not
qualify Windows lifecycle, installer/update rollback, real-model quality, external-pilot utility,
security assessment or production support.

## Validation

- Backend and frontend suites plus Python lint, formatting, TypeScript build and egress checks.
- Keyring migration, replacement, removal, secret-free JSON persistence and rollback tests.
- Installed-wheel lifecycle, recovery, integrity and real macOS Keychain set/read/delete receipt.
