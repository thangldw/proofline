# Proofline v1.1.0

Proofline v1.1.0 turns evidence-backed ADRs into a concrete product story: preserve the exact
source version and span that justified a decision, then surface when that evidence changes. It is
an experimental, non-production-qualified release for approved, recoverable local test data.

## Stale-decision workflow

- `proofline demo stale-decision` creates an approved ADR, changes its cited requirement, reports
  that the decision requires review, and writes an isolated evidence package.
- `proofline check-decisions` provides a read-only CI gate over stored decisions and opens SQLite
  with `mode=ro`; missing or corrupt evidence fails closed.
- The demo produces a human-readable `report.html`, canonical `evidence.zip`, and a content-free
  `decision-health.json` receipt.

## Open Decision Evidence Package

- Published DEP v1 as Draft 2020-12 JSON Schema with conformance vectors and stable verifier error
  expectations.
- Documented immutable-major versioning and offline verification rules so other implementations
  can consume the format.
- Added a reproducible synthetic benchmark receipt for ingest, package build, verification,
  archive size, and peak Python memory.

## Product and documentation

- Reframed the README and public project page around an ADR tied to exact requirement evidence.
- Added the stale-decision GIF, readable report preview, CI contract, benchmark results, and open
  format links.
- Moved the Merkle DAG, provenance model, and production boundaries into deeper documentation.

## Verification

The release gate runs Python and web tests, lint and formatting checks, the production web build,
credential-free evaluation gates, clean-wheel smoke tests, installed-artifact qualification, and
artifact checksums before publishing.

## Limitations

Integrity does not establish authenticity or source trust. This release has no package signature
trust model, representative-corpus production qualification, real Windows lifecycle receipt,
signed/notarized installers, external pilot outcome, real-model quality report, security
qualification, or production support commitment.
