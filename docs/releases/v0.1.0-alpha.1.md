# Proofline v0.1.0-alpha.1

This is Proofline's first pre-alpha release. It is intended for local development,
evaluation with synthetic or approved data, and early design-partner testing. It is not a stable
or production-supported release.

## Highlights

- Immutable source versions and exact source-span provenance across ingestion, memories, search,
  answers, export, backup, and deletion.
- Deterministic and provider-backed governed memories for decisions, assumptions, constraints,
  and alternatives with correction, reversible review states, audit history, and repair lineage.
- SQLite FTS5 retrieval, optional embeddings, reciprocal-rank fusion, source/indexed-time scopes,
  bounded grounded answers, fail-closed citations, and visible ranking diagnostics.
- React evidence console with source health, memory review, exact evidence navigation, deletion
  impact, and metadata-only model-run inspection.
- Credential-free extraction, Unicode/current-version retrieval, grounded-answer, platform-smoke,
  browser E2E, backup/recovery, deletion, migration, and external-egress regression gates.

## Assets

- Python wheel and source distribution for the local API/CLI.
- Optimized, unhosted web application archive for integrators. It requires a same-origin static
  host/reverse proxy that forwards `/api` and `/health` to the Proofline API; it is not a standalone
  server. A source checkout using `make dev-api` plus `make dev-web` remains the proven UI setup.
- `SHA256SUMS` covering every uploaded build artifact.

Verify downloaded files before use:

```bash
sha256sum --check SHA256SUMS
```

Run the API/CLI wheel in an isolated environment:

```bash
python3 -m venv .venv-proofline
.venv-proofline/bin/pip install ./proofline-0.1.0a1-py3-none-any.whl
.venv-proofline/bin/proofline --version
.venv-proofline/bin/proofline seed
.venv-proofline/bin/proofline serve
```

The API binds to `127.0.0.1:8000` by default. Do not expose it to an untrusted network.

## Important limitations

- The API has no authentication or multi-user authorization and binds to loopback by default.
- Proofline has not completed the repository Security plugin scan or an independent security audit.
- Real-model comparison and external pilot go/no-go metrics remain open; synthetic scores are not
  product-quality claims.
- Portable JSON import, scalable vector indexing, reranking, Windows verification, desktop
  packaging, cloud sync, and production support are not included.
- SQLite backups may contain sensitive source content and private staged retry inputs; file mode
  `0600` is not encryption.

See the root README, security policy, changelog, and backup/recovery guide before using approved
data. Report suspected vulnerabilities privately according to `SECURITY.md`.
