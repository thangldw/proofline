# Proofline v0.11.0

Proofline v0.11.0 makes the local application runnable from one installed artifact. The Python
wheel now contains the reviewed web build, so `proofline serve` starts the API and UI without a
separate web download or Node.js runtime.

## Added

- The wheel and source distribution include the built Vite UI and serve it from `/` by default.
- `proofline serve --no-web` retains an explicit API-only mode.
- `--web-dir` remains available for integrators who need to supply a reviewed replacement build.
- A deterministic bundle-sync command and release check prevent stale or mismatched web assets from
  entering a Python artifact.
- The local release path installs the wheel into a clean environment, starts its executable on a
  dynamic port, checks bundled UI and API health, then proves graceful shutdown before tagging.
- Alpha release criteria and the current support boundary are documented explicitly.

## Run

```bash
python3 -m venv .venv-proofline
.venv-proofline/bin/pip install proofline-0.11.0-py3-none-any.whl
.venv-proofline/bin/proofline serve --port 0 --data-dir .proofline-data
```

Open the loopback port from the emitted readiness JSON. Node.js is not needed at runtime.

## Limitations

This remains a pre-alpha local distribution. Windows verification, production qualification,
signed installers, external real-model evaluation, and partner pilot evidence remain open.
