# Proofline v0.14.15

Proofline v0.14.15 introduces an experimental native desktop shell while retaining the same
evidence-first local API and bundled web application. It remains a pre-alpha release.

## Included

- A Tauri v2 shell starts a target-specific PyInstaller sidecar on loopback port `0` and navigates
  the native webview only after structured readiness.
- Application state lives in the operating-system application-data directory and provider secrets
  continue to use the operating-system keyring profile.
- Closing the shell writes a private lifecycle marker; the sidecar drains Uvicorn and removes both
  readiness and shutdown markers.
- Local build commands create current-platform installers. The macOS build produces a `.app` and
  `.dmg`; the Windows configuration includes MSI/NSIS targets for a future real-Windows build.
- Python source archives explicitly exclude generated desktop/Node build trees.

## Validation

- Python lifecycle suite covers file-requested graceful shutdown.
- PyInstaller frozen binary publishes readiness, serves `/health`, exits zero and cleans markers.
- Tauri debug and release profiles compile on macOS arm64 and the release profile creates both
  application and disk-image bundles.

## Boundaries

The macOS bundle is ad-hoc signed, not Apple-notarized. Windows installer build, Credential Locker
behavior and lifecycle still need a receipt from a real Windows environment. There is no updater
or qualified rollback channel. Real-model comparison, external pilot evidence, encrypted-retention
operations and production support remain open. This release is not production-ready.
