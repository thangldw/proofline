# Proofline v0.8.0

`v0.8.0` adds an explicit, no-overwrite portable merge workflow for non-empty databases.

First create a content-free preview:

```bash
proofline import snapshot.json --preview-merge
```

Review its target counts, incoming counts, deterministic ID map and `preview_sha256`, then apply
that exact plan:

```bash
proofline import snapshot.json --merge --preview-sha256 <digest>
```

Every incoming source, version, memory, evidence row, model run, audit event and ingestion job gets
a deterministic new ID. All foreign keys, model lineage, audit targets, current-version pointers,
and immutable source identity hashes are rewritten together. Existing target rows are never updated
or deleted. If target counts change after preview, the apply is rejected; any write or provenance
failure rolls back the complete merge savepoint.

Portable schema v1 remains supported. Merge imports v1 data into the backward-compatible default
local workspace; cross-workspace export selection is not part of this release.
