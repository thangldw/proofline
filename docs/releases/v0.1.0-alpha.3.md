# Proofline v0.1.0-alpha.3

This experimental pre-alpha release adds a provenance-preserving portable restore path and a
credential-free synthetic pilot simulation. It is not stable or production-supported.

## Highlights

- `proofline import` restores a verified schema-v1 JSON export into an empty initialized database
  in one transaction.
- Exported source IDs, immutable versions, governed memories, exact evidence, safe model-run
  lineage, relevant audit events, and terminal ingestion diagnostics are preserved.
- Deterministic chunks and SQLite FTS rows are rebuilt without re-running extraction; embeddings
  remain empty for an explicit later re-index.
- Migration v12 records a unique payload-hash import receipt. Migration v13 normalizes legacy
  governed-memory statuses to the reviewable status contract.
- The verifier bounds files to 256 MiB and rejects noncanonical ordering/timestamps/numbers,
  invalid enums/ranges/lineage, forged source identity, duplicate versions, and broken spans.
- A versioned simulation runs seven scripted engineering-context tasks across five invented
  personas through the local production path.

The recorded simulation completed 7/7 scripted tasks with exact resolution/expected-source
precision of 1.0 and 8 cited-source inspections versus 29 in its deterministic naive baseline.
These are synthetic scripted observations, not human usefulness, adoption, willingness-to-pay,
external-pilot, or production-performance evidence.

## Portable restore

```bash
python3 -m venv .venv-proofline
.venv-proofline/bin/pip install ./proofline-0.1.0a3-py3-none-any.whl
PROOFLINE_DATABASE_URL=sqlite:///./restored.db \
  .venv-proofline/bin/proofline import ./proofline-export.json
```

The target must contain no domain data, index rows, retry inputs, or earlier import receipt. There
is no merge, overwrite, force, or ID-remapping mode. Any failure rolls back the importer-owned
savepoint. Imported retryable diagnostics retain their historical flag, but excluded staged inputs
cannot be restored; retry fails closed with `ingestion_input_missing`.

## Assets

- Python wheel and source distribution for the local API/CLI.
- Optimized, unhosted web archive requiring a same-origin reverse proxy for `/api` and `/health`.
- `SHA256SUMS` covering every uploaded build artifact.

## Important limitations

- The API has no authentication or multi-user authorization and binds to loopback by default.
- Repository Security plugin review, real-model comparison, external human pilot metrics, Windows
  verification, reranking, scalable vector indexing, and production qualification remain open.
- The simulation does not close any human or commercial validation gate.
- Portable merge import, desktop packaging, cloud sync, and production support are not included.

See the root README and backup/recovery guide before using approved data.
