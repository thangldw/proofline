# Proofline v0.14.10

Proofline v0.14.10 turns the documented SQLite recovery procedure into an executable, reversible
installed-artifact gate.

## Verified restore

`proofline restore-backup` verifies a backup before changing the configured SQLite database. When
the target exists, a new `--rollback-output` is mandatory. The command refuses overlapping paths,
an existing rollback output and target `-wal`/`-shm`/`-journal` sidecars, then publishes the verified candidate
atomically with owner-only permissions and verifies it again.

The preserved rollback database can be supplied to the same command to reverse the restore. The
operator must stop Proofline before either direction and keep the rollback copy until application-
level validation succeeds.

## Platform receipt

The installed-wheel receipt now creates a backup, introduces a distinct newer database state,
restores the older snapshot, confirms the newer marker is absent, restores the rollback copy and
confirms the marker returns. The receipt records `backup_restore` and `rollback_restore` only after
both directions pass.

## Qualification boundary

This closes the local macOS release-artifact restore drill, not the encrypted-retention policy,
Windows, native installer, application update rollback, real-model, external-pilot, security or
production-support gates. Proofline remains experimental pre-alpha.

## Validation

- Backend restore, rollback, refusal and CLI tests.
- Full backend/frontend, lint, formatting, web build, egress and deterministic evaluation gates.
- Installed-wheel lifecycle, portability, backup restore/rollback, integrity and macOS Keychain
  receipt.
