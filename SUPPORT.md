# Support

Proofline provides best-effort support for the latest experimental pre-release only. Supported use
is a single local user with approved, recoverable test data on a developer-controlled macOS or Linux
machine. The wheel launcher and bundled web UI are the primary experiment surface.

Windows, signed native installers, automatic updates, shared workspaces, hosted deployment, mobile,
SSO, managed sync, and production workloads are unsupported until their roadmap gates close.

Use GitHub Issues for non-sensitive defects and include the version, operating system, safe steps,
expected result, and content-free diagnostics. Never attach source documents, databases, keys,
prompts, model payloads, or pilot material. Vulnerabilities follow [SECURITY.md](SECURITY.md).

Before upgrades or restore drills, create and verify a backup outside the active data directory.
Pre-alpha schemas and APIs may change; release notes document the supported migration path.
